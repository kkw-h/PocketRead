# PocketRead UI 页面规划

## 1. 文档目标

本文档用于给 UI 设计提供完整的页面范围和页面描述，供 Android / iOS 双端首版设计使用。

适用范围：

- `v1` 本地阅读器
- 平台：Android、iOS
- 数据：本地书籍、本地进度、本地设置

设计目标：

- 让用户能快速从导入进入阅读
- 让用户能稳定管理本地书籍
- 让阅读界面沉浸、克制、少干扰

## 2. 页面范围总览

`v1` 需要设计的页面和弹层如下：

1. 启动页 / 启动中状态
2. 书架页
3. 书架空状态页
4. 文件导入结果弹层
5. 重复导入处理弹层
6. 书籍详情页
7. 删除确认弹层
8. 阅读页
9. 阅读目录页
10. 阅读快捷设置面板
11. 全局设置页
12. 通用异常状态页 / 局部异常态

说明：

- `v1` 不做登录页、账号页、同步页、在线书城页
- 阅读器以滚动阅读模式为正式实现
- 设计时要优先覆盖 `TXT` 和 `EPUB` 两种内容类型

## 3. 信息架构

建议一级结构：

- 书架
- 设置

建议主流程：

1. 启动应用
2. 进入书架
3. 导入本地书籍
4. 点击书籍进入阅读
5. 从阅读页进入目录或快捷设置
6. 从书架进入书籍详情

## 4. 页面详细说明

## 4.1 启动页 / 启动中状态

### 页面目标

- 承接应用冷启动
- 完成数据库初始化、主题恢复、最近阅读状态恢复
- 在视觉上建立产品第一印象

### 页面结构

- 品牌标识
- 简洁标题 `PocketRead`
- 轻量加载提示

### 设计建议

- 不做复杂动画
- 时长应控制在极短范围内
- 视觉风格应与阅读器主题一致，避免科技感过重

### 页面状态

- 启动中
- 初始化失败

### 设计关键词

- 安静
- 克制
- 阅读感

## 4.2 书架页

### 页面目标

- 作为应用首页
- 展示本地全部书籍
- 提供导入、搜索、排序、进入阅读的主入口

### 页面结构

- 顶部导航栏
  - 页面标题
  - 搜索入口
  - 导入按钮
  - 更多菜单或排序入口
- 书架内容区
  - 书籍卡片列表或双列网格
- 底部操作区
  - 可不单独设置 tab，若只保留书架和设置，可在顶部进入设置

### 书籍卡片信息

- 封面
- 书名
- 作者
- 格式标签 `TXT / EPUB`
- 阅读进度
- 最近阅读时间
- 收藏标识
- 置顶标识

### 核心交互

- 点击书籍：进入阅读页
- 长按书籍：弹出操作菜单
- 下拉或点击按钮导入
- 搜索书名 / 作者
- 切换排序方式

### 页面状态

- 正常列表
- 搜索结果
- 无结果状态
- 导入中局部状态

### 设计建议

- 卡片要适合长书名显示
- 进度信息要弱于书名，但不能被隐藏
- 导入按钮要足够明确
- 书架应优先展示最近阅读和置顶书籍

### 设计关键词

- 有秩序
- 易浏览
- 本地藏书感

## 4.3 书架空状态页

### 页面目标

- 在无书籍时引导用户完成第一次导入

### 页面结构

- 插画或轻量图形
- 标题
- 一句说明文案
- 主按钮 `导入书籍`
- 辅助文案：支持 `TXT / EPUB`

### 页面状态

- 首次使用空状态
- 搜索无结果空状态

### 设计建议

- 首次空状态要明显鼓励导入
- 搜索空状态要和首次空状态区分

## 4.4 文件导入结果弹层

### 页面目标

- 在导入结束后反馈结果
- 展示成功、失败、重复导入情况

### 内容结构

- 导入结果标题
- 成功数量
- 重复数量
- 失败数量
- 每本书的处理结果

### 单项结果信息

- 文件名
- 识别格式
- 导入状态
- 失败原因

### 核心交互

- 关闭
- 查看已导入书籍
- 对失败项重试

### 设计建议

- 结果应清晰可扫描
- 状态颜色区分要明显，但不要刺眼

## 4.5 重复导入处理弹层

### 页面目标

- 处理用户导入重复书籍时的分支决策

### 内容结构

- 标题：检测到重复书籍
- 已存在书籍信息
- 当前导入文件信息
- 差异提示

### 操作按钮

- 跳过
- 仍然导入
- 查看现有书籍

### 设计建议

- 重点让用户理解“为什么重复”
- 不建议给过多技术字段

## 4.6 书籍详情页

### 页面目标

- 展示书籍完整元数据
- 提供管理操作入口

### 页面结构

- 顶部导航栏
  - 返回
  - 更多操作
- 头部信息区
  - 封面
  - 书名
  - 作者
  - 格式
  - 导入时间
  - 最近阅读时间
- 统计区
  - 阅读进度
  - 章节数
  - 文件大小
- 操作区
  - 继续阅读
  - 收藏 / 取消收藏
  - 置顶 / 取消置顶
  - 删除书籍

### 可展示信息

- 文件名
- 文件格式
- 编码信息（TXT）
- 封面来源（可不直出给普通用户）
- 导入状态

### 设计建议

- 详情页应兼顾“展示”与“管理”
- 主按钮始终优先为 `继续阅读`

### 设计关键词

- 安定
- 清晰
- 卡片化信息

## 4.7 删除确认弹层

### 页面目标

- 防止误删

### 内容结构

- 标题
- 书名
- 删除后影响说明

### 按钮

- 取消
- 删除

### 设计建议

- 危险操作语义清晰
- 不要出现多重确认流程

## 4.8 阅读页

### 页面目标

- 提供沉浸式阅读体验
- 支持继续阅读、滚动阅读、呼出快捷设置

### 页面结构

- 默认状态：沉浸式正文
- 点击中部：唤出阅读工具层
- 顶部工具层
  - 返回
  - 书名 / 章节名
  - 更多操作
- 正文区
  - 章节标题
  - 正文内容
- 底部工具层
  - 目录
  - 进度
  - 设置

### 交互规则

- 单击中部：显示 / 隐藏工具层
- 向上向下滚动：阅读正文
- 点击目录：进入目录页或目录抽屉
- 点击设置：拉起快捷设置面板

### 主要信息

- 当前书名
- 当前章节
- 当前阅读进度
- 正文字体与排版

### 页面状态

- 正常阅读
- 加载章节中
- 章节解析失败
- 内容为空

### 设计建议

- 正文区域要最大化
- 工具层不能长期占屏
- 夜间模式和浅色模式都要单独出稿
- 需要同时考虑 `TXT` 和 `EPUB` 的正文样式差异

### 设计关键词

- 沉浸
- 舒适
- 温和

## 4.9 阅读目录页

### 页面目标

- 展示章节目录
- 快速跳转到目标章节

### 页面结构

- 顶部栏
  - 返回
  - 目录标题
- 列表区
  - 章节名
  - 当前章节高亮
  - 已读 / 未读弱状态

### 对 EPUB 的设计要求

- 支持层级目录
- 子章节缩进显示

### 对 TXT 的设计要求

- 优先展示自动分章结果
- 若无法精准分章，也要保证列表可点击

### 设计建议

- 当前章节高亮要足够明显
- 长目录要有良好滚动定位体验

## 4.10 阅读快捷设置面板

### 页面目标

- 在不离开阅读流程的情况下高频调节阅读体验

### 面板内容

- 字号调节
- 行高调节
- 页边距调节
- 背景主题切换
- 字体切换
- 日间 / 夜间模式切换

### 设计形式建议

- 底部上拉面板
- 分组清晰
- 调节结果即时反馈到正文

### 设计建议

- 操作路径要短
- 常用项优先放前面
- 字体和主题预览要足够直观

## 4.11 全局设置页

### 页面目标

- 管理应用级设置
- 提供不在阅读页内处理的全局选项

### 页面结构

- 顶部栏
  - 返回
  - 标题
- 设置分组
  - 阅读默认设置
  - 导入相关说明
  - 关于应用

### 建议内容

- 默认主题模式
- 默认字体
- 默认字号
- 默认背景
- 应用版本
- 当前支持格式说明

### 设计建议

- 设置页要简洁
- 避免出现与阅读快捷设置重复但更复杂的交互

## 4.12 通用异常状态页 / 局部异常态

### 页面目标

- 统一处理导入失败、解析失败、读取失败、内容为空等情况

### 适用场景

- 启动失败
- 导入失败
- TXT 解码失败
- EPUB 解析失败
- 阅读页加载失败

### 设计元素

- 状态图形
- 错误标题
- 简短说明
- 重试按钮
- 返回按钮

### 设计建议

- 文案要用户可理解
- 避免暴露底层技术细节

## 5. 页面关系图

```text
启动页
  -> 书架页
    -> 导入结果弹层
    -> 重复导入弹层
    -> 书籍详情页
      -> 删除确认弹层
      -> 阅读页
    -> 阅读页
      -> 阅读目录页
      -> 阅读快捷设置面板
    -> 设置页
```

## 6. 需要 UI 输出的设计稿范围

建议至少输出以下稿件：

1. 书架页
2. 书架空状态页
3. 导入结果弹层
4. 重复导入弹层
5. 书籍详情页
6. 删除确认弹层
7. 阅读页浅色版
8. 阅读页深色版
9. 阅读目录页
10. 阅读快捷设置面板
11. 全局设置页
12. 通用异常页

## 7. 需要 UI 单独考虑的状态

每个核心页面至少应提供以下状态：

- 正常态
- 空态
- 加载态
- 异常态

重点页面额外补充：

- 书架页：搜索态、排序态
- 阅读页：工具栏展开态、工具栏收起态、夜间态
- 导入弹层：成功、失败、重复混合态

## 8. UI 设计重点提醒

- 阅读器不是普通内容 App，正文排版优先级高于装饰性
- 书架页要有“个人藏书管理”感，不要做成纯商城卡片流
- 所有重要操作应单手可达，特别是阅读页底部操作
- Android 和 iOS 可共用主视觉，但导航、返回和弹层细节要考虑平台习惯
- 字号、行高、背景切换必须保证即时预览感
- 夜间模式不是简单反色，要重新处理对比度和背景层级

## 9. 推荐输出顺序

为了让前端更快进入实现，建议 UI 按以下顺序出稿：

1. 书架页
2. 书架空状态页
3. 阅读页
4. 阅读目录页
5. 阅读快捷设置面板
6. 书籍详情页
7. 导入相关弹层
8. 设置页
9. 异常页
