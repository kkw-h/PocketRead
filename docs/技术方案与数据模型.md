# PocketRead 技术方案与数据模型

## 1. 技术目标

本方案只服务当前 `v1`：

- 平台：Android、iOS
- 数据：完全本地化
- 阅读：支持 TXT / EPUB
- 重点：稳定导入、稳定阅读、稳定续读

## 2. 选型结论

### 2.1 客户端框架

- 选型：`Flutter`
- 结论：继续使用 Flutter，统一 Android 与 iOS 开发

原因：

- 当前项目已经以 Flutter 为主方向
- 书架、阅读器、设置页三类界面适合统一跨端实现
- Android / iOS 都能复用同一套阅读、解析、持久化逻辑

### 2.2 状态管理

- 选型：`flutter_riverpod`
- 结论：使用 Riverpod，不使用 BLoC 作为首版主状态方案

原因：

- 适合按功能拆分 provider，天然适合 `bookshelf / reader / importer / settings`
- 对异步状态、缓存、依赖注入支持更直接
- 测试友好，便于把解析器和仓储层做独立测试

约束：

- `v1` 不使用 `riverpod_generator`
- `v1` 直接使用手写 provider 和 `Notifier / AsyncNotifier`

### 2.3 路由

- 选型：`go_router`
- 结论：使用 `go_router`

原因：

- 适合声明式路由
- 后续若增加从最近阅读进入指定书籍，也能平滑扩展
- 与 Flutter Router API 对齐，后续维护成本低

建议页面结构：

- `/bookshelf`
- `/book/:bookId`
- `/reader/:bookId`
- `/reader/:bookId/toc`
- `/settings`

### 2.4 本地数据库

- 选型：`Drift + SQLite`
- 结论：首版数据库选 `Drift`，不再使用原规划中的 `Isar`

原因：

- `v1` 的核心数据天然是关系型结构
- 一本书对应多个章节
- 一本书对应一条阅读进度
- 一本书对应多条导入记录
- 全局设置与按书覆盖设置存在清晰关系
- Drift 的类型安全 SQL、迁移能力、流式监听更适合书架查询、排序、过滤和后续 schema 演进

技术约束：

- 数据库运行在后台 isolate
- 统一使用 Drift 管理结构化数据
- 不混用 `shared_preferences` 保存核心设置，避免双写

### 2.5 文件选择与权限

- 文件选择：`file_picker`
- 权限处理：`permission_handler`
- 本地路径：`path_provider`

结论：

- 导入走系统文件选择器
- 应用自己的缓存、封面和解析产物放到 app 沙盒目录
- 文件访问权限单独封装为 `PermissionService`

### 2.6 TXT 解码方案

- 编码检测：`flutter_charset_detector`
- 编码转换：`charset_converter`

结论：

- 先读取字节
- 做编码检测
- 如果检测失败，按优先级回退尝试：
  - `utf-8`
  - `gb18030`
  - `gbk`
  - `utf-16`
  - `iso-8859-1`
- 解码后统一做换行归一化和非法字符清洗

原因：

- 根目录 TXT 样本是大体积文本，且 `file` 命令显示存在非标准 UTF-8 和 `CRLF` 特征
- 如果不先解决编码问题，后续分章、目录和进度定位都会不稳定

### 2.7 EPUB 解析方案

- 元数据与打包结构解析：`epubx`
- HTML 渲染：`flutter_widget_from_html_core`

结论：

- 用 `epubx` 提取标题、作者、封面、目录、spine 顺序和章节原始 XHTML
- 用 `flutter_widget_from_html_core` 渲染章节内容

原因：

- `epubx` 已提供 EPUB 核心对象模型，适合先把 EPUB 结构读出来
- `flutter_widget_from_html_core` 对常见 HTML/CSS 支持较完整，足够覆盖 `v1` 文本型 EPUB 场景
- 比直接上 WebView 更容易统一字体、边距、背景、主题和阅读设置

### 2.8 阅读器展示方案

- 结论：`v1` 只实现左滑翻页阅读模式

原因：

- 左滑翻页需要处理定位、重排和字体变更后的恢复复杂度
- 当前 `v1` 核心是先打通导入、阅读、续读闭环
- 翻页模式通过章节字符 offset 统一 TXT / EPUB 的进度恢复

实现要点：

- 根据当前屏幕尺寸、字号、行高、页边距和字体实际排版高度生成分页
- 翻页保存 `chapter_offset` 定位，字号或主题重排后按章节内字符位置恢复
- 书籍总进度按已读字符量估算，避免长短章节差异导致百分比跳变
- EPUB 首版以解析后的正文文本参与分页，避免 HTML 章节在多页场景被截断

阅读器建议结构：

- `ReaderScaffold`
- `ReaderContentView`
- `ReaderQuickMenu`
- `ReaderTocView`
- `ReaderProgressController`

## 4. 数据模型设计原则

- `Book` 只承担书籍级元数据，不混入阅读定位细节
- `BookChapter` 承担章节和目录定位的基础单位
- `ReadingProgress` 独立存储，避免高频更新书籍表
- `ReaderPreference` 分成全局设置与按书覆盖两层
- `ImportRecord` 独立记录导入行为和失败原因，便于排错

## 5. 数据模型

### 5.1 books

用途：书架主表，保存书籍元数据。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | text | 主键，UUID |
| format | text | `txt` / `epub` |
| title | text | 书名 |
| author | text | 作者 |
| description | text nullable | 简介 |
| source_file_path | text | 原始文件路径 |
| local_file_path | text | 应用接管后的本地路径 |
| file_name | text | 原始文件名 |
| file_ext | text | 扩展名 |
| file_size | integer | 文件大小 |
| file_hash | text | 文件哈希，用于去重 |
| mime_type | text nullable | MIME 类型 |
| cover_image_path | text nullable | 封面本地路径 |
| cover_source_type | text nullable | `epub_embedded` / `generated` / `none` |
| charset_name | text nullable | TXT 检测到的编码 |
| language | text nullable | 语言 |
| total_chapters | integer | 章节总数 |
| total_words | integer nullable | 字数估算 |
| is_favorite | integer | 是否收藏 |
| pinned_at | integer nullable | 置顶时间戳，空表示未置顶 |
| import_status | text | `pending` / `ready` / `failed` |
| parse_version | integer | 解析版本号 |
| created_at | integer | 创建时间戳 |
| updated_at | integer | 更新时间戳 |
| last_read_at | integer nullable | 最近阅读时间 |
| deleted_at | integer nullable | 软删除时间，可选 |

建议索引：

- `idx_books_file_hash`
- `idx_books_last_read_at`
- `idx_books_pinned_at`
- `idx_books_title`

### 5.2 book_chapters

用途：统一保存 TXT / EPUB 的章节信息。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | text | 主键，UUID |
| book_id | text | 外键，关联 books.id |
| chapter_index | integer | 章节顺序 |
| title | text | 章节标题 |
| source_type | text | `txt` / `epub` |
| href | text nullable | EPUB 章节相对路径 |
| anchor | text nullable | EPUB 锚点或 TXT 章节锚点 |
| start_offset | integer nullable | TXT 章节开始字符偏移 |
| end_offset | integer nullable | TXT 章节结束字符偏移 |
| plain_text | text nullable | 纯文本缓存 |
| html_content | text nullable | EPUB 原始或清洗后的 HTML |
| word_count | integer nullable | 本章字数 |
| level | integer | TOC 层级，默认 1 |
| parent_id | text nullable | 层级目录父节点 |
| created_at | integer | 创建时间戳 |
| updated_at | integer | 更新时间戳 |

约束：

- `book_id + chapter_index` 唯一

建议索引：

- `idx_book_chapters_book_id_index`
- `idx_book_chapters_book_id_parent_id`

### 5.3 reading_progress

用途：保存每本书唯一的一条当前阅读进度。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | text | 主键，UUID |
| book_id | text | 外键，唯一 |
| current_chapter_index | integer | 当前章节序号 |
| chapter_offset | integer nullable | TXT 当前字符偏移 |
| scroll_offset | real nullable | 当前滚动偏移 |
| progress_percent | real | 全书阅读百分比 |
| locator_type | text | `txt_offset` / `epub_anchor` / `epub_scroll` |
| locator_value | text nullable | 具体定位值 |
| started_at | integer nullable | 首次打开时间 |
| last_read_at | integer | 最近阅读时间 |
| total_reading_minutes | integer | 累计阅读分钟数 |
| updated_at | integer | 更新时间戳 |

说明：

- `locator_type + locator_value` 用于在不同格式下恢复进度
- `scroll_offset` 只作为辅助恢复，不作为唯一定位依据

### 5.6 import_records

用途：记录导入行为、失败原因和去重结果。

| 字段 | 类型 | 说明 |
| --- | --- | --- |

| file_name | text | 文件名 |

## 6. 推荐的 Dart 实体划分

建议把数据库行模型和领域实体分开：

- 数据库表：Drift Table
- 领域实体：Plain Dart Object
- 页面状态：Riverpod State

建议实体：

- `BookEntity`
- `BookChapterEntity`
- `ReadingProgressEntity`
- `ReaderPreferencesEntity`
- `BookReaderOverrideEntity`
- `ImportRecordEntity`

## 7. 关键业务规则

### 7.1 去重规则

优先级：

1. `file_hash`
2. `title + author + file_size`
3. 用户手动确认

### 7.2 TXT 分章规则

建议顺序：

1. 正则识别常见章标题
2. 回退到固定长度分段
3. 无法分章时生成单章正文

### 7.3 阅读进度保存时机

- 章节切换时
- 页面进入后台时
- 页面退出时
- 翻页停止超过固定时间时

### 7.4 删除规则

删除书籍时需要同步删除：

- `book_chapters`
- `reading_progress`
- `book_reader_overrides`
- 关联封面文件

保留：

- `import_records`

## 8. 根目录样本对模型的影响

### TXT 样本

`《唐砖》（精校版全本）作者：孑与2.txt`

说明：

- 文件体积较大
- 存在 `CRLF`
- 编码不应假定为 UTF-8

因此需要：

- `books.charset_name`
- `book_chapters.start_offset / end_offset`
- `reading_progress.chapter_offset`

### EPUB 样本

`电影教师_青城无忌.epub`

说明：

- 需要提取封面
- 需要解析目录和章节结构

因此需要：

- `books.cover_image_path`
- `book_chapters.href`
- `book_chapters.anchor`
- `book_chapters.level / parent_id`

## 9. 首版不建议做的数据模型

以下模型暂不进入 `v1`：

- 用户账号
- 云同步记录
- 笔记
- 划线
- 听书记录
- PDF 页面定位
